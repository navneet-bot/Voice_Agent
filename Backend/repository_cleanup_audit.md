# Repository Cleanup Audit

Date: 2026-05-15

This audit is Phase 10 of the platform migration. It is intentionally non-destructive: no files are deleted, moved, renamed, or quarantined in this phase.

## Purpose

- Identify obsolete, duplicate, scratch, and local-runtime files.
- Protect production voice, websocket, campaign, transcript, recording, tenant, and telephony paths from accidental cleanup.
- Create a reviewable deprecation trail before any future quarantine or removal work.

## Safe Deprecation Policy

1. Audit only.
2. Trace dependencies, imports, runbooks, deployment scripts, and operator workflows.
3. Assign an owner and capture replacement behavior in maintained tests or documentation.
4. Quarantine with rollback in a later phase.
5. Delete only after release validation, backups, and explicit approval.
